HOW IT WORKS

Three Pillars of Reliable RAG

Traditional RAG retrieves and generates blindly. Self-RAG adds intelligence at every step of the pipeline.

01

Retrieval

Decides whether to retrieve at all. General questions are answered directly from the LLM. Only document-specific queries trigger FAISS vector search, keeping latency low and relevance high.

  • Adaptive retrieval gate
  • FAISS top-k search
  • Relevance filtering
02

Self-Evaluation

Two LLM judges run after every answer: IsSUP checks whether claims are grounded in the retrieved context, and IsUSE verifies the answer actually addresses the question.

  • IsSUP grounding check
  • IsUSE usefulness check
  • Strict quote-only revision
03

Generation

Only after the self-evaluation loop passes does the pipeline emit the final answer. If not useful, the query is rewritten and the pipeline retries automatically up to 3 times.

  • Grounded answer generation
  • Automatic query rewriting
  • Up to 3 retry cycles
