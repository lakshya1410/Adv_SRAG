How It Works

Three Pillars of Reliable RAG

Traditional RAG retrieves and generates blindly. Self-RAG adds intelligence at every step of the pipeline.

01

Retrieval

Decides whether retrieval is needed at all. General questions can be answered directly, while document-specific prompts trigger FAISS vector search for relevant chunks.

  • Adaptive retrieval gate
  • FAISS top-k search
  • Relevance filtering
02

Self-Evaluation

Two judges run after answer generation. IsSUP checks whether claims are grounded in context, and IsUSE verifies the answer actually addresses the user question.

  • IsSUP grounding check
  • IsUSE usefulness check
  • Strict quote-only revision
03

Generation

The system only returns a final answer after the evaluation loop passes. If the answer is not useful, it rewrites the query and retries automatically up to three times.

  • Grounded answer generation
  • Automatic query rewriting
  • Up to 3 retry cycles
